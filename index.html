Komponen Sistem Kontrol Kapal

Galeri komponen utama yang digunakan dalam sistem kontrol di atas kapal niaga modern.

Thermocouple (Tipe K)

Sensor suhu gas buang & suhu tinggi. Range: -200°C s/d 1750°C. Output: mV (efek Seebeck).

RTD Pt100 / Pt1000

Sensor suhu presisi tinggi. Bearing, JCW, minyak lumas. Akurasi ±0.1°C. Output: Ω (resistansi).

Pressure Transmitter

Mengukur tekanan bahan bakar, lumas, udara, steam. Output: 4-20 mA. Teknologi: strain gauge.

Coriolis Mass Flow Meter

Akurasi ±0.1%. Standar emas bunkering. Mengukur mass flow rate, densitas, suhu sekaligus.

Pneumatic Control Valve

Final control element. Globe/butterfly valve + pneumatic actuator + positioner. Fail-open / fail-close.

PID Controller

Algoritma kontrol terpopuler. >95% loop kontrol industri menggunakan PID. Auto-tuning capability.

PLC (Siemens / Allen Bradley)

Komputer industri khusus. IEC 61131-3: Ladder, FBD, ST. Scan cycle 1-100 ms. Digunakan BWTS, ballast, cargo.

Auto-Synchronizer

Menyinkronkan generator sebelum paralel ke busbar. 4 syarat: V, f, phase sequence, phase angle sama.

Minggu01

Pengantar Sistem Kontrol & Instrumentasi Kapal

Introduction to Marine Control Systems — Sejarah, Klasifikasi, dan Elemen Dasar
📚 Materi Inti

Definisi dan Urgensi Sistem Kontrol

Sistem kontrol di atas kapal adalah sekumpulan perangkat yang mengatur proses fisik kapal secara otomatis. Tanpa sistem kontrol andal, kapal modern tidak dapat beroperasi aman dan efisien. Regulasi SOLAS mensyaratkan sistem alarm dan kontrol otomatis untuk kapal UMS (Unattended Machinery Space).

  • Sejarah: dari kontrol manual (1900) → pneumatik (1940) → elektronik (1960) → DCS/PLC (1980) → AI/IoT (2000–kini)
  • OPEN-LOOP: Output tidak diukur, tidak ada koreksi otomatis. Contoh: timer pompa ballast.
  • CLOSED-LOOP: Output diukur dan diumpanbalikkan ke input untuk koreksi. Contoh: governor mesin induk.
  • Elemen dasar: Sensor/Transmitter → Controller → Actuator → Process → (feedback)
⚙️ Aplikasi di Kapal

Identifikasi Sistem Open-Loop vs Closed-Loop

Analisis sistem nyata di kamar mesin simulator.

  1. Amati dan gambar skema sistem pompa bilga manual/timer di simulator.
  2. Identifikasi: 'input', 'proses', dan 'output' dalam sistem tersebut.
  3. Diskusikan: apa yang terjadi jika level bilga terus naik melebihi kapasitas pompa?
  4. Usulkan bagaimana mengkonversi menjadi closed-loop dengan menambahkan sensor level.
💡 Fakta Historis

Kapal pertama dengan notasi UMS dari Lloyd's Register adalah MV Eurybates (1964). Sejak itu, revolusi automasi mengubah dunia pelayaran selamanya.

🌐 Perspektif Internasional

It has altered the ship's structure and function, the crew composition and training..."— Rossi, V.G. (1970), dikutip dalam Stefani, A. (2022). An Introduction to Ship Automation and Control Systems. Witherby Publishing Group.

  2. Sebutkan 5 sistem kontrol closed-loop di kamar mesin kapal niaga modern beserta fungsinya!
  3. Mengapa SOLAS mensyaratkan sistem kontrol dan alarm otomatis pada kapal UMS? Jelaskan dari perspektif teknis dan keselamatan!

📚 Sensor Tekanan & Level
  • Bourdon Tube: Mekanik, tidak perlu listrik. Untuk gauge lokal panel kamar mesin.
  • Strain Gauge Transmitter: Output 4-20 mA. Dasar pressure transmitter modern.
  • DP Transmitter: Differential pressure → level cairan. Rumus: h = P / (ρ × g).
  • Ultrasonic Level: Non-contact, cocok cairan korosif/kental (HFO, sludge).
  • Float Switch: On/off sederhana. Level alarm HIGH/LOW pada tangki.
🔢 Standar Sinyal 4-20 mA

4 mA = 0% (nilai minimum). 20 mA = 100% (nilai maksimum). 4 mA bukan 0 mA! — Ini memungkinkan deteksi kabel putus (0 mA = fault).

⚙️ Praktikum: Kalibrasi RTD & DP Level Calculation

Kalibrasi RTD Pt100

  1. Siapkan dry-block calibrator pada 25°C, 50°C, 75°C, 100°C.
  2. Masukkan Thermocouple-K dan RTD Pt100 secara bersamaan.
  3. Catat pembacaan masing-masing vs nilai standar.
  4. Hitung error: Error% = (Terukur - Standar)/Standar × 100%
  5. Tentukan apakah perlu penggantian sesuai toleransi manual.

Kalkulasi Level dari DP

Tangki ballast H=8m, air laut ρ=1025 kg/m³, DP transmitter baca 24.6 kPa:

h = P / (ρ × g)
h = 24600 / (1025 × 9.81)
h = 2.447 meter
Level = 2.447/8 × 100% = 30.6%

Soal: Apa yang berubah jika tangki diisi air tawar (ρ=1000 kg/m³)?

✏️ Soal Latihan
  1. RTD Pt100 menunjukkan 138.51 Ω. Dengan R₀=100Ω dan α=0.00385/°C, hitung suhu bearing! Apakah aman jika batas alarm HIGH = 80°C?
  2. Pressure transmitter HFO, range 0-10 bar, menunjukkan sinyal 12 mA. Berapa tekanan aktual?
  3. Mengapa 4-20 mA lebih dipilih dari 0-10V untuk instalasi kapal? Berikan 3 alasan teknis!

📚 Kontrol Viskositas HFO & Gas Analyzer

Sistem Kontrol Viskositas

  • HFO dipanaskan hingga viskositas optimal: 10-17 cSt pada titik injeksi.
  • Loop: Viscosimeter → PID Controller → Steam/Hot Water Valve → Heat Exchanger → HFO → kembali ke viscosimeter.
  • Terlalu kental: atomisasi buruk, deposit jelaga, keausan injektor.
  • Terlalu encer: bocor di pompa, smearing injektor.

Exhaust Gas Analyzer

  • O₂ optimal: 2-5% (terlalu tinggi = excess air, boros bahan bakar)
  • CO tinggi = pembakaran tidak sempurna → kurangi excess air
  • NOx: MARPOL Annex VI Tier III max untuk Emission Control Areas (ECA)
  • SOx: Global cap 0.50% sulfur sejak 1 Januari 2020
✏️ Soal Latihan

  2. Data gas buang: O₂=8.2%, CO₂=10.1%, CO=450 ppm. Analisis kondisi pembakaran dan berikan rekomendasi tuning mesin!
  3. Jelaskan mengapa Coriolis meter lebih direkomendasikan dari turbine meter untuk bunkering HFO!

📚 Tuning Ziegler-Nichols

Metode Ultimate Gain (Osilasi)

  1. Matikan aksi I dan D. Naikkan Kp perlahan hingga sistem berosilasi kontinu (Ultimate Gain = Ku).
  2. Catat periode osilasi: Pu (detik).
  3. Hitung parameter menggunakan tabel Z-N:
PID: Kp = 0.6×Ku | Ti = 0.5×Pu | Td = 0.125×Pu
PI: Kp = 0.45×Ku | Ti = 0.83×Pu
P: Kp = 0.5×Ku
⚠️ Integral Windup

Saat actuator sudah di limit (fully open/close), integrator terus menumpuk → respons sangat lambat saat kondisi berubah. Kontroler modern memiliki anti-windup protection.

⚙️ Simulasi PID — JCW Temperature Control

Langkah Simulasi (MATLAB Simulink atau Python)

  1. Buat model plant: G(s) = 1/(10s+1) × e^(-2s) (first-order + delay)
  2. Pasang P saja (Kp=2). Amati: ada steady-state error? Berapa nilainya?
  3. Tambah I (PI, Ti=20s). Offset hilang? Ada overshoot?
  4. Tambah D (PID, Td=2s). Bandingkan settling time dan overshoot.
  5. Dokumentasikan step response curve masing-masing konfigurasi.
✏️ Soal Latihan
  1. Sistem kontrol tekanan steam boiler: Kp=3, Ti=45s. Setpoint 7 bar, actual 6.3 bar. Hitung output controller saat t=0 (integral=0)!
  2. Dari tuning Z-N diperoleh Ku=5.0 dan Pu=40s. Hitung Kp, Ti, Td untuk mode PID!
  3. Mengapa aksi D tidak digunakan untuk kontrol level ballast tank? Jelaskan dari karakteristik sinyal proses!

🔴 Fail-Safe Philosophy — KRITIS!

Posisi fail-safe (fail-open/fail-close) ditentukan berdasarkan analisis HAZOP. Katup bahan bakar boiler = Fail-Close (FC) karena lebih aman memotong bahan bakar saat power hilang. Bypass cooling water = Fail-Open (FO) agar mesin tidak overheat.

📚 Control Valve & Positioner
  • Globe Valve: Plug naik-turun. Linear atau equal-percentage. Ideal untuk throttling presisi.
  • Butterfly Valve: Disk berputar. Aliran besar, hemat ruang. Kurang linear, tidak ideal throttling halus.
  • Positioner (Smart): Menerima 4-20 mA → atur tekanan udara ke aktuator → posisi valve tepat. Modern: partial stroke test, valve signature, diagnostik mandiri.
  • Cv (Flow Coefficient): Pastikan valve bekerja 20-80% pembukaan untuk kontrol optimal.
  • Instrument Air: Harus kering, bersih, bebas minyak. Dew point ≤-20°C. Kontaminasi = penyebab utama masalah aktuator pneumatik.
⚙️ Praktikum: Kalibrasi Pneumatic Control Valve
  1. Lepas sinyal 4-20 mA dari controller, hubungkan signal generator portable.
  2. Berikan 4 mA → verifikasi valve 0% (FC: fully closed).
  3. Berikan 12 mA → verifikasi 50%. Jika deviasi >2%, kalibrasi positioner.
  4. Berikan 20 mA → verifikasi 100% (fully open).
  5. Uji seat leakage: dengan valve tertutup, ukur kebocoran (cc/menit atau ANSI Class).
  6. Partial stroke test: gerak ke 10% dan kembali 0% — pastikan valve tidak lengket.
✏️ Soal & Referensi
  1. Aktuator pneumatik spring-return, spring force 800 N, diameter diafragma 150 mm. Udara 4.5 bar. Hitung gaya yang dihasilkan tekanan udara dan verifikasi apakah cukup melawan spring!
  2. Jelaskan perbedaan smart positioner modern vs positioner lama! Apa keunggulan untuk predictive maintenance?

Minggu06

Sistem Kontrol Mesin Induk (Main Propulsion Engine)

Remote Control, Electronic Governor, Sequential Start & Safety Interlocks
📚 Remote Control & Governor
  • Bridge Control System: Telegraph elektronik memungkinkan kontrol mesin induk langsung dari anjungan. Control transfer: Bridge → ECR → Local (hanya satu aktif).
  • Governor Mekanik (Centrifugal): Bobot berputar, gaya sentrifugal mengontrol rack bahan bakar. Sederhana, andal, mesin lama.
  • Governor Hidrolik: Pilot valve + power piston + oli bertekanan. Respons lebih halus.
  • Governor Elektronik (Woodward, Heinzmann): Magnetic pickup RPM → PID microprocessor → servo actuator → fuel rack. Akurasi ±0.25% RPM.
  • Overspeed Protection: WAJIB terpisah dari governor utama. Trip otomatis jika RPM ≥ 110-115% nominal.
📚 Sequential Starting & Interlocks

Sekuens Start Mesin Induk

  1. Pre-start checks: tekanan lumas ✓, suhu JCW dalam range ✓, turning gear bebas ✓.
  2. Aktifkan pompa pre-lube → tunggu tekanan OK.
  3. Buka katup udara start → mesin berputar.
  4. Injeksikan bahan bakar saat putaran cukup → mesin hidup.
  5. Pompa pre-lube off setelah tekanan lumas normal dari pompa utama.
🔴 Interlock Kritis

Turning gear masih terpasang → No Start. Low lube oil pressure → Auto Shutdown. High JCW temperature → Slowdown/Shutdown. Engine room bilge HIGH → No Start.

⚙️ Troubleshooting: Governor Hunting

Gejala: RPM naik-turun ±15 RPM terus-menerus

  1. Kemungkinan Kp terlalu tinggi → kurangi gain governor secara bertahap.
  2. Periksa fuel linkage: binding atau backlash memperburuk respons.
  3. Verifikasi kalibrasi speed sensor — noise sinyal menyebabkan respons tidak stabil.
  4. Cek hydraulic oil condition (governor hidrolik): kavitasi atau kontaminasi air.
  5. Review auto-tune jika governor elektronik — jalankan auto-tune ulang.
✏️ Soal Latihan
  1. Mesin induk MAN B&W 6S50ME-C: MCR 9480 kW pada 127 RPM. Overspeed trip di 15% atas MCR RPM. Hitung RPM trip!
  2. Jelaskan mengapa overspeed protection harus merupakan sistem TERPISAH dan INDEPENDEN dari governor utama!
  3. Apa yang dimaksud 'Load Program' pada sistem starting mesin induk modern? Mengapa penting untuk mencegah thermal overload?
Minggu07

Sistem Kontrol Boiler

Combustion Control · Three-Element Drum Level · Steam Pressure · Safety Interlocks
📚 Combustion Control & 3-Element Level

Cross-Limiting Combustion Control

  • Beban naik: tambah udara DULU, baru bahan bakar → cegah asap hitam (black smoke)
  • Beban turun: kurangi bahan bakar DULU, baru udara → cegah kebakaran unburnt fuel
  • O₂ Trim Control: fine-tuning rasio udara-bahan bakar berdasarkan bacaan O₂ real-time

Fenomena Swell & Shrink

  • SWELL: Beban naik → tekanan steam turun → bubble dalam air bertambah → level naik SEMU. Single-element controller keliru menutup feedwater!
  • 3-Element Solution: Steam flow (feedforward) + Feedwater flow (feedback inner) + Drum level (setpoint koreksi).
📚 Boiler Safety Interlocks
  • LOW WATER LEVEL SHUTDOWN: Level turun di bawah minimum absolute → burner mati otomatis. Mencegah tube overheating/collapse.
  • HIGH PRESSURE CUTOUT: Tekanan steam melebihi maksimum → burner mati. Safety valve mekanik = lapisan terakhir.
  • FLAME FAILURE: UV flame detector tidak mendeteksi nyala dalam <4 detik → bahan bakar diputus. Mencegah furnace explosion.
  • LOW ATOMISING STEAM: Tekanan steam atomising rendah → burner tidak bisa mengatomisasi HFO dengan baik → shutdown.
  • LOSS OF FD FAN: Tanpa udara forced draft → pembakaran tidak sempurna → shutdown.
💡 3-Element Control Summary

Elemen 1: Level Drum (koreksi). Elemen 2: Steam Flow (feedforward). Elemen 3: Feedwater Flow (inner feedback). Ketiganya bekerja bersama untuk stabilitas level saat beban transien.

✏️ Soal Latihan
  1. Jelaskan fenomena swell and shrink! Mengapa single-element control tidak memadai untuk boiler besar?
  2. Gambarkan diagram blok three-element control! Identifikasi feedforward path dan feedback path!
  1. Boiler beroperasi 7 bar, kapasitas 3 ton/jam. Beban naik mendadak menjadi 4.5 ton/jam. Jelaskan bagaimana 3-element control merespons vs single-element!
  2. Flame detector UV tidak merespons saat burner dinyalakan. Sebutkan 5 kemungkinan penyebab!

Minggu09

Sistem Kontrol Turbin Uap & Gas

Steam & Gas Turbine Speed Governing, Vibration Monitoring & Protection Systems
📚 Governing Turbin & Protection
  • Hydraulic Governing: Governor sentrifugal → pilot valve → actuator → steam admission valve.
  • Electronic Governing: RPM sensor → microprocessor PID → electro-hydraulic servo → steam valve. Presisi lebih tinggi, integrasi dengan PMS.
  • Nozzle Control: Mengaktifkan sejumlah nozzle berdasarkan beban → efisiensi lebih baik pada beban parsial.
  • DROOP: Penurunan RPM proporsional terhadap kenaikan beban. Diperlukan untuk load sharing antar turbin/generator paralel.

Sistem Trip Turbin (WAJIB)

  • OVERSPEED TRIP: Mekanik (centrifugal bolt) atau elektronik, independen dari governor. Trip pada 110-112% RPM nominal.
  • LOW LUBE OIL PRESSURE: Kerusakan bearing dalam hitungan detik.
  • HIGH VIBRATION: Accelerometer → trip saat melampaui 7.1 mm/s (ISO 10816 Zone D).
  • HIGH EXHAUST TEMPERATURE: Blade turbin gas terancam di atas batas.
⚙️ Soal Latihan
  1. Turbo-generator beroperasi 3600 RPM (60 Hz). Overspeed trip = 10% di atas nominal. Hitung RPM trip! Jika akselerasi 500 RPM/s, berapa waktu tersedia untuk trip?
  2. Data vibration monitoring: Jan 3.2 mm/s → Feb 4.1 mm/s → Mar 6.2 mm/s. Berdasarkan ISO 10816, tindakan apa yang harus diambil?
  3. Jelaskan perbedaan prinsip governing antara turbin uap dan mesin diesel! Mengapa overspeed pada turbin lebih berbahaya?
  4. Apa itu critical speed pada turbin dan mengapa operasi berkelanjutan di kecepatan tersebut berbahaya?

Minggu10

Power Management System (PMS) & Distribusi Listrik

Auto-Synchronizer · Load Sharing · Load Shedding · Blackout Prevention
📚 PMS & Auto-Synchronizer
  • Fungsi PMS: Auto-start/stop DG berdasarkan beban, paralleling otomatis, load sharing, load shedding, alarm/proteksi listrik.
  • 4 Syarat Paralel Generator: (1) Tegangan sama, (2) Frekuensi sama, (3) Phase sequence sama, (4) Phase angle ≈ 0.
  • AVR (Auto Voltage Regulator): Atur arus eksitasi → tegangan stabil. Saat paralel, AVR kontrol reactive power (kVAR) sharing.
  • Governor untuk kW sharing: Setelah paralel, governor atur active power (kW) sharing proporsional kapasitas masing-masing DG.
  • DEAD BUS START: DG pertama ke busbar yang mati total — tidak perlu sinkronisasi.
🔴 Load Shedding Priority

Stage 1 (85%): WARNING alarm. Stage 2 (90%): Shed HVAC, electric heater. Stage 3: Shed bow thruster, cargo crane. PROPULSI & KESELAMATAN tidak boleh diputus!

✏️ Soal & Referensi

Soal Latihan

  1. Kapal memiliki 3 DG @ 800 kW. DG1 & DG2 beroperasi, total beban 1720 kW. Bow thruster (300 kW) request start. PMS response apa? Hitung beban share optimal jika DG3 ikut beroperasi!
  2. Jelaskan perbedaan load shedding vs preferential trip dalam konteks PMS!
  3. SOLAS mensyaratkan emergency generator auto-start dalam 45 detik. Sistem apa yang harus disuplai segera? Sebutkan minimal 8!
  4. Jika dua generator diparalelkan saat phase angle berbeda 60°, apa yang terjadi? Jelaskan konsekuensinya!

Minggu11

Kontrol Mesin Bantu: Purifier, Refrigerasi & Kompresor

Auxiliary Machinery Automation — Auto-Desludging · Variable Speed Refrigeration · Starting Air
📚 Purifier Otomatis & Refrigerasi

Auto-Desludging Purifier

  • Interval waktu yang diprogram (mis. 60 menit) → sistem otomatis membuka water chamber → bowl terbuka → sludge terbuang (<2 detik) → bowl menutup kembali.
  • Water transducer: deteksi air dalam minyak bersih → alarm HIGH WATER → alirkan ke dirty tank.
  • Suhu feed HFO dikontrol pada 95-98°C untuk pemisahan optimal.

Variable Speed Refrigeration

  • Inverter-driven compressor: kecepatan disesuaikan beban pendinginan aktual → hemat energi 20-40%.
  • Thermostat elektronik → liquid line solenoid valve → kontrol suhu ruang.
  • HP/LP cutout: safety switch matikan kompresor jika tekanan di luar batas.
⚙️ Troubleshooting & Soal

Troubleshooting: High Water in Clean Oil (Purifier)

  1. Periksa kondisi water seal dalam bowl — jika kurang, tambahkan sealing water.
  2. Cek suhu feed HFO — harusnya 95-98°C. Suhu rendah = pemisahan buruk.
  3. Verifikasi kapasitas throughput tidak melebihi rating purifier.
  4. Inspeksi disc stack dan gravity disc — tentukan gravity disc yang tepat.

Soal Latihan

  1. Gambarkan flowchart logika kontrol proses auto-desludging purifier!
  2. Apa perbedaan purifier vs clarifier? Sistem kontrol mana lebih kompleks?
  3. Kompresor udara start 2-tahap tidak bisa mencapai 30 bar, hanya 22 bar. Sebutkan 4 kemungkinan penyebab!
Minggu12

Perlengkapan Dek, Steering Gear & Autopilot

Follow-Up / Non-Follow-Up · Autopilot PID · Windlass Tension Control
📚 Steering Gear Control
  • NON-FOLLOW-UP (NFU): Kemudi bergerak selama tuas ditekan, berhenti saat dilepas. Tidak ada feedback posisi. Untuk steering darurat.
  • FOLLOW-UP (FU): Posisi kemudi mengikuti handle secara proporsional. Ada feedback dari follow-up transmitter. Standar navigasi normal.
  • SOLAS Req.: Dari 35° satu sisi ke 35° sisi lain: MAKSIMUM 28 detik pada MCR.
  • AUTOPILOT: Gyrocompass mengukur heading → dibandingkan setpoint → PID controller → rudder order → steering gear. Gain adaptif terhadap kecepatan dan kondisi cuaca.
  • Auto-Mooring Winch: Load cell mengukur tegangan tali → winch otomatis merespons perubahan pasang surut mempertahankan tension setpoint.
⚙️ Pengujian SOLAS & Soal

Prosedur Pengujian Steering Gear (Pre-Departure)

  1. Uji 35° kanan → 35° kiri menggunakan main steering system — catat waktu (<28 detik).
  2. Uji auxiliary steering system secara terpisah.
  3. Uji komunikasi anjungan ↔ steering gear room.
  4. Verifikasi alarm: low oil level, high oil temperature, phase failure.
  5. Catat dalam steering gear log book dan ditandatangani perwira.

Soal Latihan

  1. Gambarkan diagram lengkap sistem telemotor elektrik steering gear!
  2. Steering tidak merespons dari anjungan (autopilot) tapi merespons manual dari steering room. Di mana kemungkinan lokasi gangguan?
  3. Apa itu weather helm dan bagaimana parameter autopilot disesuaikan saat cuaca buruk?
Minggu13

BWTS & Pemantauan Polusi Lingkungan

Ballast Water Treatment · OWS 15ppm · Oil Content Monitor · Scrubber SOx Control
📚 BWTS & OWS Control

Ballast Water Treatment System (UV-Based)

  • Air ballast → reaktor UV → dosis ≥45 mJ/cm² → DNA mikroorganisme rusak.
  • Sensor: UV Intensity (pastikan dosis cukup) + UV Transmittance (turbiditas air).
  • Jika UVT rendah (air keruh) → sistem meningkatkan dosis UV atau reject ballasting.
  • Flow proportional dosing: pompa dosis disesuaikan laju aliran secara otomatis.

OWS — Oil Content Monitor (OCM)

  • MARPOL Annex I: buangan bilga ke laut ≤15 ppm (di luar special area).
  • OCM (infrared/light scattering): mengukur konsentrasi minyak kontinyu.
  • Jika >15 ppm → 3-way solenoid valve otomatis mengalihkan ke bilge holding tank.
  • ORB Digital Logger: mencatat tanggal, waktu, posisi GPS, volume, kandungan minyak — TAMPER PROOF.
✏️ Soal & Referensi

Soal Latihan

  1. Mengapa UV Transmittance (UVT) merupakan parameter kritis dalam UV-BWTS? Apa yang terjadi jika UVT rendah?
  2. OWS mengolah 2 m³/jam air bilga. Setelah separator: 18 ppm. Setelah polishing: 8 ppm. Apakah boleh dibuang ke laut? Jelaskan berdasarkan MARPOL Annex I!
  3. Apa yang dimaksud tampering pada sistem OCM? Bagaimana desain modern mencegahnya?
  4. Jelaskan perbedaan open-loop dan closed-loop scrubber! Kapan closed-loop menjadi mandatory?

  • MARPOL Annex I — Regulations for Prevention of Pollution by Oil.

⚠️ EEMUA 191 — Alarm Flood

Maksimum 1 alarm per 10 menit dalam kondisi normal operasi. Alarm flood (terlalu banyak alarm sekaligus) justru BERBAHAYA — operator tidak bisa mengidentifikasi alarm mana yang kritis!

Predictive Maintenance Tools

  • Vibration Analysis: Spectrum analysis mendeteksi bearing defect, imbalance, misalignment sebelum kegagalan.
  • Oil Analysis (OLA): Kandungan partikel logam mengidentifikasi keausan internal mesin.
  • Thermography IR: Hotspot pada panel listrik, motor, pipa — deteksi dini koneksi longgar.
  • PTK TCMS: PT Pertamina Trans Kontinental — TCMS (Transko Condition Monitoring System) memantau kapal tanker secara digital real-time dari darat.
✏️ Soal Latihan
  1. Jelaskan perbedaan alarm vs shutdown! Berikan 3 contoh parameter yang memiliki alarm level DAN shutdown level!
  2. Mengapa alarm flood justru berbahaya dari perspektif keselamatan? Hubungkan dengan EEMUA Publication 191!
  3. Data vibration DG: Jan 3.2 → Feb 4.1 → Mar 6.2 mm/s. Berdasarkan ISO 10816, tindakan apa yang harus diambil? Justifikasi dengan Zone A/B/C/D!
  4. Jelaskan prinsip dasar analisis vibration untuk mendeteksi bearing defect! Apa yang dimaksud bearing defect frequency?

Minggu15

DCS, PLC & Cyber Security Maritim

Distributed Control Systems · PLC Programming · Modbus · IMO Cyber Risk Management
📚 DCS, PLC & Protokol
  • DCS Hierarki: Field Level (sensor, actuator) → Control Level (PLC, local panel) → Supervisory Level (HMI) → Management Level (reporting).
  • Hot Standby Redundancy: Controller cadangan selalu mirroring data controller utama — switchover dalam milidetik tanpa gangguan proses.
  • Ring Topology: Jika jalur komunikasi utama gagal, data mengalir melalui jalur alternatif.
  • PLC Scan Cycle: Input Scan → Program Execution → Output Update. Scan time: 1-100 ms.
  • IEC 61131-3 Languages: LD (Ladder), FBD (Function Block), ST (Structured Text), IL (Instruction List), SFC (Sequential Function Chart).
  • Modbus TCP/IP: Protokol paling umum menghubungkan PLC ↔ HMI ↔ DCS melalui Ethernet di kapal.
🔐 Cyber Security Maritim
🔴 Kasus Nyata: NotPetya 2017

Ransomware NotPetya menyerang Maersk Line — seluruh jaringan IT lumpuh. Kerugian diperkirakan $300 juta. Kapal tidak bisa melapor, menerima pesanan, atau beroperasi normal selama berhari-hari.

  • ANCAMAN: Malware via USB, unauthorized remote access via VSAT, social engineering (phishing) target awak kapal.
  • Network Segmentation: IT network (email, internet) HARUS dipisahkan dari OT network (sistem kontrol kapal) menggunakan firewall dan DMZ.
  • IMO MSC-FAL.1/Circ.3: Identifikasi aset kritis → penilaian risiko → deteksi insiden → respons dan pemulihan → audit berkala.
  • ISM Code (seit 2021): Manajemen Risiko Cyber WAJIB diintegrasikan dalam Safety Management System kapal.

Soal Latihan

  1. Jelaskan perbedaan DCS vs SCADA! Sistem mana lebih banyak di kapal niaga modern?
  2. Apa itu air gap sebagai strategi cyber security? Apakah masih praktis untuk kapal modern yang butuh shore-based monitoring?
  3. USB drive ditemukan di pantry kapal dan dimasukkan ke komputer navigasi. Analisis risiko, prosedur yang seharusnya, dan cara melaporkan insiden ini!
